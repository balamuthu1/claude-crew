---
description: Interactive setup to capture your team's workflow tool preferences (ticketing, docs, comms). Writes workflow.config.md. Run once after install, before any other detect command.
---

Run directly — do not spawn a sub-agent.

## What this does

Every team uses different tools for tickets, documentation, and communication.
This command captures your preferences so every agent uses the right system —
creating tickets in YOUR tool, linking docs in YOUR platform, and referencing
the right channels.

Reads `workflow.config.md` if it already exists and shows current values.
You can re-run at any time to update preferences.

---

## Step 1 — Check for existing config

Read `workflow.config.md` if present. If found, show the current values and ask:
```
workflow.config.md already exists. Current settings:
  [show current values]

Update it? [y/N]
```
If N, exit. If Y (or file doesn't exist), continue to Step 2.

---

## Step 2 — Ticket / issue tracking system

Ask:
```
Which ticket / issue tracking system does your team use?

  1) Jira           (Atlassian — requires jira-cli)
  2) Linear         (Linear app)
  3) GitHub Issues  (built into your GitHub repo)
  4) Azure DevOps   (Azure Boards)
  5) Shortcut       (formerly Clubhouse)
  6) Asana
  7) Trello
  8) Notion         (Notion databases as tasks)
  9) Other / none

Enter number (or type the name):
```

For choice 1 (Jira), also ask:
```
  Jira project key (e.g. PROJ, APP, BE):
  Jira base URL (e.g. https://yourcompany.atlassian.net):
  Is the Jira CLI installed? (jira --version) [y/N]:
```

For choice 2 (Linear), also ask:
```
  Linear team name or identifier:
  Do you use Linear's Git integration (auto-links branches)? [y/N]:
```

For choice 3 (GitHub Issues), also ask:
```
  GitHub repo (owner/repo):
  Do you use GitHub Projects (kanban board)? [y/N]:
```

For choice 4 (Azure DevOps), also ask:
```
  Azure DevOps organization URL:
  Project name:
```

For choice 5 (Shortcut), ask:
```
  Shortcut workspace slug:
```

```
  4) Markdown files in the repo
  5) GitBook
  6) SharePoint / Teams Wiki
  7) Readme.io / ReadTheDocs
  8) Obsidian (shared vault)
  9) Other / none

Enter number (or type the name):
```

For choice 1 (Confluence), also ask:
```
  Confluence base URL (e.g. https://yourcompany.atlassian.net/wiki):
  Space key for technical docs (e.g. ENG, TECH):
```

For choice 2 (Notion), also ask:
```
  Notion workspace name:
  Is there a specific Notion page/database for PRDs or technical specs? [y/N]
  If yes, paste the URL or ID:
```

For choice 4 (Markdown in repo), also ask:

```
  Docs directory path (e.g. docs/, .docs/, wiki/):
```

## Step 7 — Write workflow.config.md

Write `workflow.config.md` in the project root:

```markdown
# Claude Crew — Workflow Config
# Generated by /detect-workflow — edit manually to update

## Ticket System

ticket_system: <jira|linear|github-issues|azure-devops|shortcut|asana|trello|notion|other>
ticket_id_format: <e.g. PROJ-123 or #456>

# Jira (if ticket_system: jira)
jira_project_key: <KEY>
jira_base_url: <https://...>
jira_cli_available: <true|false>

# Linear (if ticket_system: linear)
linear_team: <team-name>
linear_git_integration: <true|false>

# GitHub Issues (if ticket_system: github-issues)
github_repo: <owner/repo>
github_projects: <true|false>

# Azure DevOps (if ticket_system: azure-devops)
azure_org_url: <https://...>
azure_project: <project-name>

# Shortcut (if ticket_system: shortcut)
shortcut_workspace: <slug>

## Documentation

docs_platform: <confluence|notion|google-docs|markdown|gitbook|sharepoint|readme|obsidian|other>

# Confluence (if docs_platform: confluence)
confluence_url: <https://...>
confluence_space: <SPACE_KEY>

# Notion (if docs_platform: notion)
notion_workspace: <name>
notion_prd_page: <url-or-id>

# Markdown (if docs_platform: markdown)
docs_directory: <docs/>

## Communication

comms_tool: <slack|teams|discord|email|other>

# Slack (if comms_tool: slack)
slack_engineering_channel: <#channel>
slack_incidents_channel: <#channel>

# Teams (if comms_tool: teams)
teams_team: <name>
teams_channel: <name>

## Process

sprint_length_weeks: <1|2|3|4|kanban>
```

Only write fields relevant to the chosen tools. Leave out sections for tools not in use.

---

Show a summary of what was written:
```
✓ workflow.config.md written.

  Ticket system   : <system>
  Docs platform   : <platform>
  Communication   : <tool>
  Sprint length   : <N weeks>

All agents will now use these preferences.

Recommended next steps:
  /detect-gitflow           ← configure git branching conventions
  /detect-<profile>-stack   ← configure your team's tech stack

Re-run /detect-workflow at any time to update.
```
