# PRD Writing Workflow

1. Spawn `prd-author` with the feature description
2. Draft problem statement and goals
3. Write functional requirements with edge cases
4. Write acceptance criteria in Given/When/Then
5. Define success metrics with baselines
6. Spawn `metrics-analyst` to validate the metrics definition
7. Output: complete PRD + metrics event schema
