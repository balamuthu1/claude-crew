# Automation Framework Workflow

1. Read `qa.config.md` for declared test framework
2. Spawn `automation-engineer` to assess current framework setup
3. Produce: page objects/app actions, test fixtures, CI integration config
4. Apply best practices: `data-testid` selectors, explicit waits, test isolation
5. Output: framework setup files + example test demonstrating patterns
