Read the project's build files to detect the actual architecture, then write
a `claude-crew.config.md` in the project root.

## Step 1 — Detect Platform

Look for:
- `build.gradle.kts` or `build.gradle` → Android
- `Package.swift` or `*.xcodeproj` or `Podfile` → iOS
- `package.json` with `react-native` → React Native
- `pubspec.yaml` with `flutter` → Flutter

If both Android and iOS exist, platform is `both`.

## Step 2 — Read Android Build Files

If Android detected, read these files (if they exist):
- `app/build.gradle.kts` or `app/build.gradle`
- `gradle/libs.versions.toml`
- `buildSrc/` or `build-logic/` directories
- Any `*.gradle.kts` file

Extract:
- **minSdk** / **targetSdk** / **compileSdk**
- **Dependencies** — scan for these patterns:

| Dependency pattern | Config value |
|---|---|
| `hilt-android` | di: hilt |
| `koin-android` or `insert-koin` | di: koin |
| `dagger:dagger` (no hilt) | di: dagger2 |
| `compose-bom` or `compose.ui` | ui: compose |
| `fragment-ktx` without compose | ui: xml (check layout files to confirm) |
| `kotlinx-coroutines` | state: coroutines-flow |
| `rxjava2` or `rxandroid` | state: rxjava2 |
| `rxjava3` | state: rxjava3 |
| `retrofit` | networking: retrofit |
| `ktor-client` | networking: ktor |
| `room-runtime` | storage: room |
| `realm-kotlin` | storage: realm |
| `coil` | images: coil |
| `glide` | images: glide |
| `mockk` | mocking: mockk |
| `mockito` | mocking: mockito |
| `navigation-compose` | navigation: navigation-compose |
| `navigation-fragment` | navigation: navigation-fragment |

- **Architecture Pattern** — infer from source structure:
  - Look for `presentation/viewmodel/` or `*ViewModel.kt` → mvvm
  - Look for `*Presenter.kt` or `*Contract.kt` → mvp
  - Look for `*Reducer.kt` or `*Intent.kt` or `*Store.kt` → mvi
  - Look for `*Controller.kt` in presentation layer → mvc

## Step 3 — Read iOS Build Files

If iOS detected, read:
- `Package.swift`
- `Podfile` and `Podfile.lock`
- `*.xcconfig`
- `project.pbxproj` (extract IPHONEOS_DEPLOYMENT_TARGET)
- Any `Info.plist` for minimum OS version

Extract:
- **Deployment target**
- **Dependencies**:

| Dependency | Config value |
|---|---|
| `Swinject` | di: swinject |
| `Resolver` | di: resolver |
| No DI library | di: manual |
| `RxSwift` | state: rxswift |
| `Combine` (or no 3rd-party reactive) | state: combine (check for Publisher usage) |
| `Alamofire` | networking: alamofire |
| `Moya` | networking: moya |
| No network library | networking: urlsession |
| `RealmSwift` | storage: realm |
| CoreData model files exist | storage: coredata |
| `SwiftData` imports | storage: swiftdata |
| `Quick`, `Nimble` | test-framework: quick-nimble |

- **Architecture Pattern** — infer from source structure:
  - `*ViewModel.swift` + `ObservableObject` or `@Observable` → mvvm
  - `*Presenter.swift` + `*View.swift` protocol → viper or mvp
  - `*Reducer.swift` or `ComposableArchitecture` import → tca
  - `*Store.swift` with `Reducer` type → tca

## Step 4 — Detect Mixed/Legacy State

Look for signs of in-progress migration:
- Both `compose-bom` AND layout XML files → ui: mixed
- Both `kotlinx-coroutines` AND `rxjava` → state: mixed (note in legacy-notes)
- Both `SwiftUI` AND `UIKit` view controllers → ui: mixed

## Step 5 — Scan Source for Confirmation

Quickly scan source files to confirm what was found in build files:
- `find . -name "*.kt" | head -20` → look at 2-3 files for patterns
- `find . -name "*.swift" | head -20` → look at 2-3 files for patterns
- `find . -name "*.xml" -path "*/layout/*" | head -5` → confirms XML UI if present

## Step 6 — Write claude-crew.config.md

Write the config file to the project root with all detected values filled in.
Use comments to explain anything uncertain.

Format (fill in detected values — do NOT leave template comments):

```markdown

# Auto-generated by /detect-arch on [DATE]
# Edit to correct anything the detector got wrong.

platform: [detected value]

## State Management

## Dependency Injection

## Networking
networking: [detected value]

## Local Storage
storage: [detected value]

## Image Loading (Android)
images: [detected value or none]

## Testing Framework

```
